# 更新日志

## [未发布] - 2026-04-25

### 新增
- 食材数据库：管理员可在后台维护家庭共享食材库（名称 + 默认单位），支持创建、删除
- 菜品表单结构化食材：添加/编辑菜品时可从食材库选择食材并设置用量，支持在线新建食材
- 订单食材汇总：管理员点击订单可查看每道菜的食材明细及全单食材总量汇总
- 预约管理页完整表单：加入菜单时支持图片上传（预览确认后上传）、结构化食材选择、类型/分类选择
- 轮播图编辑功能：支持对已有轮播图进行内容编辑
- 轮播图1:1比例：上传预览和首页展示统一为正方形裁剪
- 图片上传预览：选图后先预览确认（× 取消），点"确认使用"才真正上传
- 个人资料编辑页：支持修改头像（上传至云存储）和昵称
- 家庭名称管理：管理员可在资料编辑页修改家庭名称，普通成员仅可查看
- 新增 `user` 云函数：负责用户注册/查询/更新昵称头像/家庭名称等操作

### 修复
- 修复预约"加入菜单"表单中分类选择器无响应的 bug（picker 需包裹整个 field 视图）
- 修复表单内输入框右侧紧贴屏幕边缘的问题（scroll-view 增加 box-sizing: border-box）
- 修复轮播图首页展示比例异常（改用 aspectFill 模式）
- 修复 .gitignore 未排除云函数 node_modules 导致 git add 暂存万余文件

## [未发布] - 2026-04-13

### 重构
- 后端完全迁移至微信云开发，彻底去除 FastAPI 服务器和 SQLite 数据库
- 新增 6 个云函数：banners / categories / dishes / orders / reservations / admin
- 云数据库替代 SQLite，使用 MongoDB 风格文档存储，订单明细嵌入订单文档
- 云存储支持菜品图片和轮播图上传
- 前端 API 调用从 wx.request HTTP 改为 wx.cloud.callFunction，移除 BASE_URL 配置
- 用户身份由云函数通过 cloud.getWXContext().OPENID 自动获取，无需前端传递 user_id
- 新增小程序内管理员页面（菜品管理、轮播图管理、数据初始化），替代原有 Web 管理界面
- 管理员通过 openid 白名单鉴权
