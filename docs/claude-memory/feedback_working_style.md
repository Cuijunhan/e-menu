直接实施，不要反复询问用户确认。

**Why:** 用户明确说过"后续不需要再询问我，直接给我重构完成整个项目"。用户对代码有掌控力，不需要每步都确认。

**How to apply:** 设计阶段可以按流程提问（brainstorming），但一旦用户说"实施吧"或"直接做"，就不再询问，直接完成所有任务后汇报结果。例外：需要用户提供外部信息（如环境 ID、openid）时才需要说明。
