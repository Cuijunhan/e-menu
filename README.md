# 家庭点餐小程序

一个用于家庭内部使用的微信点餐系统，包含微信小程序前端和微信云开发后端（云函数 + 云数据库 + 云存储）。

---

## 技术架构

- **前端**：微信小程序（原生开发）
- **后端**：微信云开发（云函数 × 6，无需自维护服务器）
- **数据库**：微信云数据库（MongoDB 风格）
- **存储**：微信云存储（菜品图片、轮播图）

---

## 云函数列表

| 云函数 | 职责 |
|--------|------|
| `banners` | 轮播图查询 |
| `categories` | 主分类/子分类查询（支持 food/coffee/drink 三类） |
| `dishes` | 菜品列表/随机/详情（food_dishes / coffee_dishes / drink_dishes） |
| `orders` | 订单创建/查询/删除/状态更新 |
| `reservations` | 预约 CRUD |
| `admin` | 管理员操作（菜品/轮播图/食材/预约/订单管理） |
| `user` | 用户信息（注册/查询/更新昵称头像/家庭名称） |

## 部署步骤

### 1. 开通云开发

在微信开发者工具中点击「云开发」按钮，开通并记录 **环境 ID**。

### 2. 配置环境 ID

将 `app.js` 中的 `YOUR_ENV_ID` 替换为实际环境 ID：

```js
const ENV_ID = 'cloud1-xxxxxxxx'  // 替换为你的环境 ID
```

### 3. 创建云数据库集合

在云开发控制台手动创建以下集合（权限建议）：

| 集合名 | 权限设置 |
|--------|---------|
| `banners` | 所有用户可读，仅管理员可写 |
| `food_categories` | 所有用户可读，仅管理员可写 |
| `coffee_categories` | 所有用户可读，仅管理员可写 |
| `drink_categories` | 所有用户可读，仅管理员可写 |
| `food_dishes` | 所有用户可读，仅管理员可写 |
| `coffee_dishes` | 所有用户可读，仅管理员可写 |
| `drink_dishes` | 所有用户可读，仅管理员可写 |
| `ingredients` | 所有用户可读，仅管理员可写 |
| `orders` | 仅创建者可读写 |
| `reservations` | 仅创建者可读写 |
| `users` | 仅创建者可读写 |
| `families` | 仅创建者可读写 |

> 云数据库权限可在控制台 → 数据库 → 集合 → 权限设置中配置。

### 4. 部署云函数

在微信开发者工具中，右键 `cloudfunctions/` 下每个云函数目录 → **上传并部署（云端安装依赖）**。

### 5. 获取并配置管理员 openid

1. 在小程序中打开个人中心，会自动获取 openid 并存入 Storage
2. 或在云开发控制台 → 云函数 → `admin` → 测试，传入 `{ action: "getOpenid" }` 获取 openid
3. 将获取到的 openid 填入 `cloudfunctions/admin/index.js`：

```js
const ADMIN_OPENIDS = ['你的真实openid']
```

4. 重新部署 `admin` 云函数

### 6. 初始化默认数据

管理员登录后，进入**个人中心 → 管理后台 → 初始化默认数据**，写入初始分类和菜品。

**局域网访问配置：**

后端服务监听 `0.0.0.0:8000`，支持局域网内其他设备访问。小程序需配置本机IP地址：

```js
// utils/api.js
const BASE_URL = "http://192.168.1.172:8000";  // 替换为你的本机IP
```

获取本机IP：Windows 运行 `ipconfig`，macOS/Linux 运行 `ifconfig`。

### 3. 验证运行

浏览器访问以下地址确认服务正常：

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:8000 | API 根节点（返回 JSON） |
| http://127.0.0.1:8000/docs | 自动生成的 Swagger 接口文档 |
| http://127.0.0.1:8000/admin | 后台管理界面（添加菜品/分类） |

## 主要功能

- **三类分类系统** — 饭菜/咖啡/酒水三个独立分类体系，各自管理子分类和菜品
- **点菜** — 按分类浏览菜品，加入购物车下单，点击菜品查看详细介绍（含食材、做法）
- **首页快捷入口** — 点击 COOKING/COFFEE/WINE 直接跳转到对应分类
- **随机推荐** — 随机生成今日菜单，三种动画特效，智能过滤已加入的菜品
- **历史订单** — 查看订单记录，支持取消/删除订单
- **想吃什么** — 预约菜单上没有的菜，支持编辑和删除预约
- **个人中心** — 头像昵称编辑、家庭名称管理（管理员专属）
- **后台管理（小程序内）** — 菜品增删改、分类管理、轮播图编辑、食材数据库、预约处理、订单管理
- **食材数据库** — 维护家庭共享食材库，菜品绑定结构化食材，订单可查看食材汇总
- **轮播Banner** — 首页展示精选推荐，支持裁剪预览后上传，支持编辑
- **自定义TabBar** — 玻璃态效果底部导航栏

## 预约功能说明

首页新增「想吃什么」入口，可以预约目前菜单上没有的菜：

1. 输入**菜名**（必填）
2. 粘贴**抖音或小红书链接**（可选，方便参考做法）
3. 填写**备注**，如口味偏好、特殊要求（可选）
4. 提交后可查看历史预约记录，状态分为**待处理** / **已处理**
5. 点击预约记录可进入详情页，支持编辑和删除（已处理的预约无法修改）

后端 API：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /reservations | 提交预约 |
| GET | /reservations?user_id=1 | 查询预约列表 |
| PUT | /reservations/{id} | 更新预约内容 |
| DELETE | /reservations/{id} | 删除预约 |
